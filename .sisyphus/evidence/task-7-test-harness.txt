Task 7 — test harness upgrade and verification
Date: 2026-04-08

Package updates
- @playwright/test: 1.58.2 -> 1.59.1
- vitest: 4.1.2 -> 4.1.3
- tsx: 4.21.0 (already at target)

Harness/config changes
- Added scripts/typecheck.ts to make `pnpm typecheck` reproducible with
  Next.js 16.2.2 + TypeScript 6.0.2.
- Root cause: `next typegen` generates `.next/types/validator.ts` importing
  `./routes.js`, while only `routes.d.ts` exists. The script now runs
  `next typegen`, writes `routes.js.d.ts` shims into generated type folders,
  then runs `tsc --noEmit`.
- Updated package.json `typecheck` script to `tsx scripts/typecheck.ts`.
- Updated public Playwright config to use an isolated default port (`3013`),
  pass `PORT`, `BETTER_AUTH_URL`, and `NEXT_PUBLIC_APP_URL` through the
  Playwright webServer env, and disable server reuse by default unless
  `PLAYWRIGHT_REUSE_EXISTING_SERVER=true` is explicitly set.
- Admin Playwright config and Vitest config remained compatible with the
  upgraded package APIs; no API-shape changes were needed there.

- `pnpm typecheck`: PASS
- `pnpm build`: PASS
- `pnpm test:e2e`: PASS (16 tests)
- `pnpm test:e2e:admin`: PASS (16 tests)

Environment notes
- Commands emitted the existing engine warning because the workspace is running
  Node v22.22.2 while package.json requires Node 24.x.
- Public E2E initially exposed the prior harness flaw: sharing port 3011 with a
  pre-existing process was non-reproducible. The isolated-port webServer fix
  resolved that and restored deterministic startup + seeded DB behavior.
