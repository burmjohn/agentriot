# AgentRiot implementation guidance

## Required skills and tools

- Use `shadcn` and `shadcn-ui` for component discovery, installation, and
  updates.
- Use `vercel-react-best-practices` when building or refactoring React and
  Next.js code.
- Use `frontend-design` for visible UI work that must align with `DESIGN.md`.
- Use `test-driven-development` before adding new behavior.
- Use `playwright-expert` for E2E coverage or Playwright debugging.
- Use `requesting-code-review` before landing major implementation work.
- Use Context7 and web search before changing framework, package, or CLI
  versions.

## Foundation constraints

- Keep versions pinned to the exact values documented in `VERSION-POLICY.md`.
- Treat `DESIGN.md` as the visual source of truth and keep the app dark-only.
- Treat `AgentRiot_V2_Project_Plan.md` as the product boundary source of truth.
- Do not modify `.sisyphus/plans/*.md` files.
